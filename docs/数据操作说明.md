# 数据操作说明

codeUtils库在数据操作上支持了标签转换、图像裁切、标签类别重构。

## 标签转换

标签转换目前支持LABELME、PASCAL VOC、COCO、YOLO之间的相互转换, 快速加载：

```python
from codeUtils.labelOperation import coco2yolo
```

目前已经支持12个接口, 具体接口为:

- [X] **yolo2coco**: yolo格式转coco格式, src_dir为yolo的根目录, dst_dir为coco的根目录, 输出为json文件, 同时要保障图像是一一对应的.
- [X] **yolo2labelme**: 指定names(ID2NAME), 将yolo标签转换为labelme标签文件.
- [X] **yolo2voc**: 指定names(ID2NAME), 将yolo标签转换为PASCAL VOC标签文件.
- [X] **labelme2yolo**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, names参数同上, dst_dir为yolo格式的根目录.
- [X] **labelme2voc**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, dst_dir为PASCAL VOC格式的根目录.
- [X] **labelme2coco**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, names参数同上, dst_dir为COCO格式的根目录.
- [X] **voc2labelme**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, dst_dir为labelme格式的根目录.
- [X] **voc2yolo**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, names参数同上, dst_dir为yolo格式的根目录.
- [X] **voc2coco**: lbl_dir与img_dir是一一对应的, img_dir为空时, 默认使用标签同文件夹, names参数同上, dst_dir为yolo格式的根目录.
- [X] **coco2yolo**: 指定coco格式标注文件[支持多个]或者所在的文件夹, 指定转换后的yolo标签存放地址即可.
- [X] **coco2labelme**: 指定coco格式标注文件[支持多个]或者所在的文件夹, 指定转换后的labelme标签存放地址即可.
- [X] **coco2voc**: 指定coco格式标注文件[支持多个]或者所在的文件夹, 指定转换后的Pascal VOC标签存放地址即可.

### yolo2coco

yolo2coco接口定义:

```python
def yolo2coco(src_dir: Path, dst_dir: Path, names: str|dict, use_link: bool = False,
              split: str = 'train', img_idx: int = 0, ann_idx: int = 0,
              year: str = "", class_start_index: Literal[0, 1] = 0) -> None: ...
```

**参数说明:**

- src_dir: yolo格式的根目录, 包含子目录images和labels.
- dst_dir: coco格式的根目录, 结果包含子目录annotations和图像子集文件夹.
- names: 类别名称映射, 键为类别ID, 值为类别名称. names支持dict、yaml文件和txt文档.
- use_link: 图像是否使用软链接, 默认为False.
- split: 图像子集名称, 默认为train.
- img_idx: 图像起始索引, 默认为0.
- ann_idx: 标注起始索引, 默认为0.
- year: 年份, 默认为空.
- class_start_index: 类别ID起始索引, 默认为0.

**调用案例:**

```python
>>> from codeUtils.labelOperation import yolo2coco
>>> yolo2coco(
...     src_dir=Path("datasets/coco128/"),
...     dst_dir=Path("datasets/coco128/coco"),
...     names={
...         0: "person",
...         1: "bicycle",
...         2: "car",
...         3: "motorcycle",
...         4: "airplane",
...         79: "toothbrush",
...     }
... )
```

### labelme2yolo

labelme2yolo的接口定义：

```python
def labelme2yolo(lbl_dir: List[Path | str] | Path | str, dst_dir: str | Path,
                 names: str|dict, img_dir: List[Path | str] | Path | str = []) -> None:
```

**参数说明：**

- lbl_dir: labelme格式的标注文件或目录, 支持多个文件或目录.
- dst_dir: yolo格式的根目录, 结果包含子目录images和labels.
- names: 类别名称映射, 键为类别ID, 值为类别名称. 支持names为dict、yaml文件和txt文档.
- img_dir: 图像文件或目录, 支持多个文件或目录. 没有指定时, 默认使用标签的地址, 即标签与图像在同一文件夹.

```python
...     dst_dir=Path("/Users/elfindan/datasets/coco128/yolo2"),
...     names={
...         0: "person",
...         1: "bicycle",
...         2: "car",
...         78: "hair drier",
...         79: "toothbrush",
...     }
... )
```

### coco2yolo

coco2yolo的接口定义：

```python
def coco2yolo(lbl_dir: List[Path|str]| Path| str, dst_dir: Path | str): ...
```

**参数说明：**

- lbl_dir: coco格式标注文件的目录或者coco格式标注文件, 支持直接指定多个coco格式标注文件.
- dst_dir: yolo格式的根目录, 结果存放在dst_dir文件夹. 只存txt文件, 不处理图像.

**调用案例：**

```python
>>> from codeUtils.labelOperation import coco2yolo
>>> coco2yolo(
...     lbl_dir=["/Users/elfindan/datasets/coco128/coco/annotations/train2026.json"],
...     dst_dir=Path("/Users/elfindan/datasets/coco128/yolo4"),
... )
```

其中ldl_dir可以是coco标注文件地址, str或者Path对象, 也可以是包含这些标签的文件夹, 默认会读取文件夹下所有的json文件, 存放的时候也是根据json文件划分子集存放, 图像不做同步.
